# پیشنهاد اجرایی خوانا برای نسخه بعدی Subnet Calculator

این سند، نسخه ساده و قابل اجرا از پیشنهادهاست تا دقیق بدانیم چه کاری، با چه اولویتی انجام شود.

---

## 1) تغییرات درخواستی شما (خلاصه سریع)

### 1. زبان پیش فرض
- زبان پیش فرض برنامه: **English**
- اگر کاربر زبان را تغییر داد، انتخابش ذخیره شود.
- اگر تنظیمی ذخیره نشده بود، برنامه همیشه با انگلیسی بالا بیاید.

### 2. دو تب بالای صفحه Subnet
- در بالای صفحه Subnet، دو تب داشته باشیم:
  - IPv4
  - IPv6
- تب پیش فرض: **IPv4**
- تب IPv6 فعال باشد و خروجی پایه تحلیل IPv6 را نمایش دهد.

### 3. پشتیبانی Prefix به دو فرم
در ورودی Prefix، هر دو حالت پشتیبانی شود:
- CIDR: مثل `24` یا `/24`
- Subnet Mask: مثل `255.255.255.0`

نکته مهم:
- اگر کاربر Mask وارد کرد، سیستم آن را به Prefix تبدیل کند.
- فقط ماسک های معتبر (contiguous) قبول شوند.
- برای ورودی اشتباه، پیام خطای دقیق نمایش داده شود.

### 4. آموزش جامع تر و غیرخشک
- آموزش از متن ساده به حالت **تعاملی** تبدیل شود.
- با فلش، رنگ بندی بیت ها، اسلایدر Prefix و مثال مرحله ای، کاربر مفهوم را بصری یاد بگیرد.

## 2) پیشنهاد دقیق UI/UX برای صفحه Subnet

### چیدمان پیشنهادی
1. AppBar
2. TabBar در بالا (IPv4 / IPv6)
3. محتوای تب IPv4:
   - فیلد IP
   - فیلد Prefix/Mask
   - دکمه Calculate
   - کارت نتایج
4. محتوای تب IPv6:
  - یک کارت معرفی
  - ورودی IPv6/Prefix
  - دکمه تحلیل
  - خروجی فشرده/کامل، Prefix و نوع آدرس

### نکته تجربه کاربری
- برای فیلد Prefix، Placeholder واضح بگذاریم:
  - Example: 24 or 255.255.255.0
- زیر فیلد ورودی یک Helper Text کوتاه نمایش داده شود.

---

## 3) پیشنهاد دقیق منطق Prefix/Mask

### سناریوهای ورودی
- اگر ورودی عددی بود: Prefix فرض شود.
- اگر ورودی شامل نقطه بود: Mask فرض شود.
- اگر با `/` شروع شد، `/` حذف و Prefix پردازش شود.

### قوانین اعتبارسنجی
- Prefix باید بین 0 تا 32 باشد.
- Mask باید IPv4 معتبر باشد.
- بیت های Mask باید contiguous باشند.
  - معتبر: 255.255.255.0
  - نامعتبر: 255.0.255.0

### خروجی داخلی واحد
- در نهایت، برای موتور محاسبه، همیشه یک مقدار Prefix داشته باشیم.
- یعنی حتی Mask هم قبل از محاسبه به Prefix تبدیل شود.

---

## 4) آموزش تعاملی: به جای متن خشک چه کنیم؟

### مدل آموزشی پیشنهادی (مرحله ای)
هر درس 3 بخش داشته باشد:
1. توضیح ساده
2. نمایش شماتیک/انیمیشن
3. تمرین کوتاه

### اجزای بصری پیشنهادی
- نوار 32 بیتی رنگی:
  - بیت های Network یک رنگ
  - بیت های Host یک رنگ دیگر
- اسلایدر Prefix:
  - با تغییر اسلایدر، تقسیم بیت ها زنده تغییر کند.
- فلش مرحله ای:
  - IP -> Mask -> Network -> Broadcast
- کارت مقایسه:
  - /24 در برابر /26

### چرا این روش بهتر است؟
- کاربر فقط متن نمی خواند؛ می بیند و تعامل می کند.
- یادگیری سریع تر و ماندگارتر می شود.
- برای کاربران مبتدی بسیار موثرتر از توضیح صرفا متنی است.

## 5) سرفصل پیشنهادی بخش آموزش (نسخه جامع)

### سطح 1: مفاهیم پایه
- IP چیست؟
- تفاوت IPv4 و IPv6
- Prefix و Subnet Mask چیست؟

### سطح 2: محاسبات
- Network Address
- Broadcast Address
- Host Range
- تعداد میزبان قابل استفاده

### سطح 3: مهارت عملی
- مثال واقعی: شبکه دفتر کوچک
- تقسیم یک /24 به چند subnet کوچک
- خطاهای رایج کاربران

### سطح 4: تمرین و آزمون
- تمرین های کوتاه با پاسخ
- Quiz 3 سوالی انتهای هر درس

## 6) نقشه اجرای پیشنهادی (کم ریسک)

### فاز 1 (سریع)
- پیش فرض زبان انگلیسی
- تب های IPv4/IPv6 در بالای صفحه
- IPv6 پایه فعال (Basic/Advanced)

### فاز 2 (عملیاتی)
- پشتیبانی کامل Prefix و Mask در یک فیلد
- اعتبارسنجی دقیق و پیام خطای واضح

### فاز 3 (ارتقای آموزش)
- اضافه شدن نمودار بیتی + فلش مرحله ای + اسلایدر
- ساخت درس های جامع تر

### فاز 4 (یادگیری پیشرفته)
- تمرین تعاملی و Quiz
- مسیر یادگیری مبتدی تا نیمه حرفه ای

اگر این مسیر را برویم:
- تجربه کاربر واضح تر و حرفه ای تر می شود.
