# نقشه راه عملی توسعه IPv6 برای Subnet Calculator

این سند، یک برنامه اجرایی مرحله‌ای برای توسعه IPv6 است که همزمان دو هدف را دنبال می‌کند:
- ساخت یک موتور IPv6 دقیق و قابل اعتماد
- افزایش درک کاربر از تفاوت IPv6 و IPv4 با تمرکز بر آموزش کاربردی

> اصل کلیدی: IPv6 فقط «ابزار محاسبه» نیست؛ باید با «آموزش قابل فهم» عرضه شود.

---

## 1) چشم‌انداز محصول

### هدف نهایی
- کاربر مبتدی بتواند مفاهیم IPv6 را در چند دقیقه بفهمد.
- کاربر فنی بتواند محاسبات ضروری IPv6 را سریع و دقیق انجام دهد.
- کیفیت فعلی بخش آموزش IPv4 حفظ و حتی بهتر شود (بدون افت).

### اصل طراحی
- `Teach + Tool` به‌جای `Tool only`
- هر قابلیت فنی IPv6 باید یک لایه توضیح ساده هم داشته باشد.

---

## 2) ستون‌های اجرایی پروژه

### ستون A: موتور IPv6
- اعتبارسنجی دقیق IPv6
- نرمال‌سازی (compressed/expanded)
- تشخیص نوع آدرس IPv6
- محاسبه پایه Prefix و Network Summary

### ستون B: UX و جریان کاربر
- ارائه سطح‌بندی شده (Basic / Advanced)
- توضیح کوتاه کنار هر خروجی
- کاهش اصطلاحات پیچیده در نمای اولیه

### ستون C: آموزش
- درس‌های کوتاه، تصویری و تعاملی
- مقایسه مستقیم IPv4 و IPv6
- تمرین‌های مرحله‌ای + Quiz

### ستون D: کیفیت و تست
- Unit Test و Widget Test کافی
- Performance قابل قبول روی گوشی میان‌رده
- صفر افت کیفیت در بخش IPv4

---

## 3) اصول مهم پیش از شروع

### 3.1 حفظ کیفیت آموزش IPv4 (الزامی)
در طول توسعه IPv6، بخش آموزش IPv4 باید:
- حذف یا کم‌اهمیت نشود
- از نظر محتوا به‌روزرسانی شود (با مثال‌های بهتر)
- در UI همچنان به‌عنوان مسیر آموزشی اصلی باقی بماند

### 3.2 عدم شلوغی بیش از حد
- از اضافه کردن امکانات enterprise در نسخه اول IPv6 خودداری شود.
- ابتدا قابلیت‌های پرکاربرد و قابل فهم ارائه شود.

### 3.3 معماری پایدار
- فیچر IPv6 مستقل در ساختار کلین اضافه شود.
- اشتراکات فنی فقط در core قرار گیرند.

## فاز 1: IPv6 MVP فنی + آموزشی پایه (5 تا 7 روز)

### هدف
ارائه IPv6 پایه به شکلی که کاربر هم «بتواند استفاده کند» و هم «بفهمد» چه اتفاقی افتاده است.

### قابلیت‌های فنی
1. Validation IPv6:
   - پشتیبانی `::`
   - بررسی تعداد گروه‌ها
   - بررسی هگز معتبر
2. Normalization:
   - نمایش Compressed
   - نمایش Expanded (8 گروه)
3. Classification:
   - Global Unicast
   - Link-local
   - Unique Local
   - Multicast
   - Loopback/Unspecified
4. ورودی `IPv6/Prefix` مثل `2001:db8::1/64`

### قابلیت‌های UI
1. تب IPv6 فعال با دو بلوک:
   - بلوک آموزشی کوتاه (Understand)
   - بلوک محاسبه (Calculate)
2. کنار هر خروجی یک توضیح یک‌خطی:
   - مثلا: `Link-local: usable only inside local link`
3. پیام خطای انسانی:
   - نه فقط invalid input، بلکه توضیح علت

### معیار تکمیل
- خروجی‌های اصلی درست باشند.
- کاربر مبتدی بدون دانش قبلی بتواند تفاوت compressed/expanded را بفهمد.

## فاز 2: ارتقای آموزش IPv6 + حفظ کیفیت IPv4 (4 تا 6 روز)

### هدف
ارتقای بخش آموزش به سطحی که تفاوت IPv4/IPv6 برای کاربر روشن شود.

### تغییرات آموزشی IPv6
1. ماژول "IPv4 vs IPv6 in 5 minutes"
2. کارت‌های مقایسه‌ای:
   - 32bit vs 128bit
   - Decimal dotted vs Hex groups
   - Broadcast (IPv4) vs Multicast (IPv6)
3. نمودار مرحله‌ای:
   - Input -> Normalize -> Classify -> Prefix Summary
4. تمرین تعاملی:
   - تشخیص نوع آدرس IPv6

### ارتقای موازی IPv4 (الزامی)
1. بازبینی محتوای آموزش IPv4 برای کیفیت نگارشی و مثال‌ها
2. اضافه کردن یک بخش "اشتباهات رایج IPv4"
3. تست تجربه کاربر:
   - اطمینان از سریع و روان بودن آموزش IPv4 پس از اضافه شدن IPv6

### معیار تکمیل
- بخش IPv4 افت کیفیت نداشته باشد.
- کاربر بتواند تفاوت IPv4/IPv6 را در یک جلسه کوتاه درک کند.

- کاربر مبتدی در Basic دچار سردرگمی نشود.
