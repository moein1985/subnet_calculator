# طرح توسعه و انتشار اپلیکیشن Subnet Calculator (اندروید - استورهای ایرانی)

## 1) هدف محصول
هدف، ساخت یک اپلیکیشن ساده، سریع و قابل اعتماد برای محاسبات شبکه است که در فاز اول برای کاربران ایرانی (دانشجویان، ادمین‌ها، پشتیبان‌ها و علاقه‌مندان شبکه) تجربه‌ای روان و آموزشی ارائه دهد و در عین حال زیرساخت فنی آن برای توسعه‌های آینده آماده باشد.

خروجی مورد انتظار:
- انتشار نسخه پایدار اندروید در کافه‌بازار و مایکت
- تجربه کاربری دو زبانه (فارسی و انگلیسی)
- معماری مقیاس‌پذیر برای اضافه شدن قابلیت‌های حرفه‌ای

## 2) چشم‌انداز فنی (Future-Proof)
برای جلوگیری از بازنویسی در آینده، معماری اپ از ابتدا لایه‌بندی شده و ماژولار طراحی می‌شود.

### تکنولوژی‌های پیشنهادی
- Flutter (پلتفرم اصلی)
- go_router برای مدیریت مسیرها و ناوبری ساختاریافته
- flutter_bloc + bloc + equatable برای مدیریت وضعیت قابل تست و مقیاس‌پذیر
- get_it (یا injectable در فاز بعد) برای Dependency Injection
- intl + flutter_localizations برای چندزبانه‌سازی استاندارد
- hydrated_bloc (اختیاری) برای نگهداری stateهای مهم مثل تنظیم زبان
- shared_preferences یا Hive برای ذخیره داده‌های سبک (تاریخچه و تنظیمات)

### ساختار پوشه‌ای پیشنهادی
- lib/core
- lib/features/subnet
- lib/features/education
- lib/features/settings
- lib/l10n
- lib/router

ساختار هر feature:
- data
- domain
- presentation

## 3) تجربه کاربری و زبان
### دو زبانه بودن (الزامی)
اپلیکیشن از فارسی و انگلیسی پشتیبانی می‌کند.

نیازمندی‌ها:
- انتخاب زبان از داخل برنامه (فارسی/English)
- پشتیبانی کامل RTL برای فارسی
- ترجمه کامل متن‌ها (بدون متن Hardcoded)
- ذخیره زبان انتخابی کاربر برای اجراهای بعدی

### اصول UX پیشنهادی
- صفحه اصلی بسیار ساده: ورودی، نتیجه، اکشن‌ها
- نمایش خطاهای انسانی و واضح (IP نامعتبر، Prefix نامعتبر، ناسازگاری ورودی)
- دکمه کپی برای فیلدهای کلیدی
- حفظ سادگی بصری برای کاربران تازه‌کار

## 4) قابلیت‌های محصول (فاز 1)
### 4.1 محاسبات IPv4
فاز اول محاسبات کامل IPv4:
- Network Address
- Broadcast Address
- Subnet Mask
- Wildcard Mask
- First Host / Last Host
- Number of Usable Hosts
- نمایش باینری (اختیاری در MVP+)

### 4.2 پشتیبانی IPv6 (وضعیت فعلی)
به دلیل ضعف دانش عمومی کاربران در IPv6، این قابلیت علاوه بر حضور در UI به صورت عملیاتی در سطح پایه ارائه شده است.

پیاده‌سازی فاز فعلی:
- تب IPv6 در صفحه اصلی
- تحلیل پایه IPv6 شامل:
  - نمایش فرم فشرده و کامل
  - نمایش Network Prefix
  - نمایش نوع آدرس
  - حالت Basic/Advanced

هدف محصولی:
- از همین نسخه، کاربر خروجی کاربردی IPv6 دریافت می‌کند
- در فاز بعد، بدون تغییر معماری، تحلیل‌های پیشرفته‌تر IPv6 اضافه می‌گردد

### 4.3 بخش آموزش (الزامی)
یک بخش مستقل با عنوان آموزش اضافه می‌شود.

محتوای پیشنهادی آموزش:
- IP چیست؟
- تفاوت IPv4 و IPv6
- Subnet Mask و Prefix Length
- Network/Broadcast/Host Range با مثال ساده
- سناریوهای واقعی: تقسیم شبکه دفتر کوچک
- اشتباهات رایج کاربران

ویژگی‌های UX بخش آموزش:
- محتوای کوتاه، بخش‌بندی‌شده و قابل اسکرول
- امکان جستجوی ساده (در فاز بعد)
- امکان بوکمارک (در فاز بعد)

## 5) ناوبری (go_router)
پیشنهاد مسیرها:
- / (خانه)
- /education (آموزش)
- /history (تاریخچه)
- /settings (تنظیمات)

نکات:
- deep link برای فاز آینده در نظر گرفته شود
- route guard برای بخش‌هایی مثل Pro در آینده قابل اضافه شدن باشد

## 6) مدیریت وضعیت (BLoC)
BLoCهای پیشنهادی:
- AppSettingsBloc: زبان، تم، تنظیمات عمومی
- SubnetCalculatorBloc: ورودی‌ها، اعتبارسنجی، خروجی IPv4
- EducationBloc: بارگذاری محتوای آموزشی
- HistoryBloc: ذخیره و بازیابی محاسبات اخیر
- FeatureFlagsBloc (اختیاری): کنترل قابلیت‌های آماده/در حال ارتقا مثل Advanced IPv6

مزایا:
- تست‌پذیری بالا
- جداسازی کامل UI از business logic
- توسعه‌پذیری برای قابلیت‌های آینده

## 7) تست و کیفیت
حداقل تست‌های ضروری:
- Unit test برای موتور محاسبه IPv4
- Unit test برای Validator ورودی‌ها
- Bloc test برای حالت‌های success/error/loading
- Widget test برای مسیرهای اصلی و سوییچ زبان

کیفیت انتشار:
- Crash-free startup
- عملکرد سریع روی دستگاه‌های میان‌رده
- بدون Permission غیرضروری
