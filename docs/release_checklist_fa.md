# چک‌لیست عملی انتشار اندروید و QA

این سند برای بستن فاز پایدارسازی و آماده‌سازی انتشار تهیه شده است.
وضعیت هر مورد را با یکی از این برچسب‌ها ثبت کنید:
- [ ] انجام نشده
- [~] در حال انجام
- [x] انجام شده
- [!] بلاک‌شده

---

## 1) پیش‌نیازهای محیط انتشار

- [ ] دسترسی DNS و اینترنت به مخازن زیر برقرار است:
  - `dl.google.com`
  - `repo.maven.apache.org`
  - `storage.googleapis.com`
- [ ] در صورت استفاده از پراکسی، تنظیمات `HTTP_PROXY` و `HTTPS_PROXY` در محیط یا Gradle اعمال شده است.
- [ ] زمان سیستم و تاریخ دستگاه توسعه صحیح است.
- [ ] فضای دیسک کافی برای build artifacts وجود دارد.

### فرمان‌های سریع بررسی شبکه (PowerShell)

```powershell
Resolve-DnsName dl.google.com
Resolve-DnsName repo.maven.apache.org
Resolve-DnsName storage.googleapis.com
```

در صورت خطای DNS، ابتدا مشکل شبکه را رفع کنید؛ بیلد release/profile بدون دسترسی به مخازن قابل تکمیل نیست.

## 4) سناریوهای رگرسیون دستی (الزامی)

### 4.1 بخش IPv4

- [ ] ورودی با prefix معتبر مثل `192.168.1.10/24`
- [ ] ورودی با subnet mask معتبر مثل `255.255.255.0`
- [ ] پوشش edge caseهای `/31` و `/32`
- [ ] کپی خروجی‌ها (network, broadcast, first/last host) صحیح است.

### 4.2 بخش IPv6

- [ ] ورودی معتبر مثل `2001:db8::1/64`
- [ ] نمایش `compressed` و `expanded` صحیح است.
- [ ] نوع آدرس (Global, Link-local, Multicast, ...) درست نمایش داده می‌شود.
- [ ] حالت `Basic/Advanced` بدون خطا کار می‌کند.
- [ ] پیام خطا برای ورودی نامعتبر انسانی و واضح است.

### 4.3 آموزش

- [ ] بخش آموزش IPv4 بدون افت کیفیت قابل استفاده است.
- [ ] بخش مقایسه IPv4/IPv6 و تمرین تعاملی درست نمایش داده می‌شود.
- [ ] ناوبری بین تب‌ها/صفحه‌ها روان و بدون glitch است.

## 5) پروفایل و کارایی

- [!] اجرای `flutter run --profile` در وضعیت فعلی بلاک‌شده (مشکل DNS مخازن)
- [ ] پس از رفع شبکه: زمان شروع برنامه و روانی اسکرول روی دستگاه واقعی ثبت شود.

### فرمان پیشنهادی

```bash
flutter run --profile -d <device-id> --trace-startup --no-resident
```

## 6) بیلد انتشار اندروید

- [!] `flutter build apk --release` در وضعیت فعلی بلاک‌شده (عدم resolve مخازن)

- [ ] در صورت نیاز: تولید AAB برای انتشار Google Play

```bash
flutter build apk --release
flutter build appbundle --release
```
